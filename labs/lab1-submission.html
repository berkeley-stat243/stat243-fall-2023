Hands-on Lab Instructions

Problem Set submission instructions:

  1. Open the qmd file in any editor you like (e.g., Emacs, Sublime, â€¦.). Use quarto preview FILE to show your rendered document live as you edit and save changes. You can put the preview window side by side with your editor, and the preview document should automatically render as you save your qmd file.
  2. Use VS Code with the following extensions: Python, Quarto, and Jupyter Notebooks. This allows you to execute and preview chunks (and whole document) inside VS Code. This is currently deebâ€™s favorite path due to how well it integrated with the Python debugger.
  3. Use RStudio (yes, RStudio), which can manage Python code and will display chunk output in the same way it does with R chunks. This path seems to work quite well and is recommended if you are already familiar with RStudio.

Steps to perform today:

  1. Clone your github repository to your development environment
  2. Create a subdirectory in your github repository with the name ps0
  3. In that subdirectory, create a quarto document (ps0.qmd) that has some simple code that creates a simple plot (you can follow this example/tutorial here)
  4. Use the quarto command line to render it into a pdf document (quarto render FILE â€“to pdf)
  5. Commit the changes to your repository (git add FILES; git commit -m MESSAGE; git push)
  6. Add another section to your quarto document (use your imagination), then preview and commit the changes
  7. Use the quarto command line to render the updated document into a pdf document
  8. Add the pdf document to the repository as well
  9. Make sure that you can log into gradescope and upload a pdf document
  10. [optional] Undo your last set of changes and regenerate the pdf file

If we finish early, We will also take todayâ€™s lab as an opportunity to get familiar with the basic use of all the 5 basic tools listed above.

For git and quarto, very basic knowledge should be sufficient for now, but for unix commands and python, the more you learn the more effective you will be at solving the problem sets (and at any computational task you take on after that). You will need to learn more advanced use of git and github towards the end of the semester when you start working with other team members on the same project.
